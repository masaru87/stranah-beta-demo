## Task 1: ワークスペース作成（機能ID: 7）

- **確認:** ヘッダー右「ワークスペース作成」ボタン → ダイアログで名前入力 → 作成 → カード追加

## Task 2: 通知センター（機能ID: 63）
- **確認1:** 全ページ共通ヘッダー右のベルアイコン（赤い未読バッジ付き）

- **確認2:** 未読行（青背景+ドット）/ 既読行の区別、「すべて既読にする」ボタン

## Task 3: GitHub連携強化（機能ID: 55）
- **ページ:** `/project/pj-1/settings`
- **確認:** 「GitHub連携」カード内に接続状態（緑ドット）、トリガー設定チェックボックス2つ

## Task 4: ドリフト検知 + 環境状態同期（機能ID: 18, 46）
- **ページ:** `/env/env-1/drift`
- **確認:** サイドバー「ドリフト検知」→ 差分テーブル（期待値=緑、実測値=赤）、「環境状態を同期」ボタン

## Task 5: リソース一覧（機能ID: 48）
- **ページ:** `/env/env-1/resources`
- **確認:** サイドバー「リソース一覧」→ 9リソースのテーブル（ステータス/名前/種別/メタデータ）

## Task 6: Service/Infra管理（機能ID: 64, 65, 17）
- **ページ:** `/env/env-1/services`
- **確認:** サイドバー「Service / Infra」→ 2タブ切替

  - **Infraタブ:** さくらクラウド月額コスト合計 + 6リソースのスペック・コスト表

## Task 7: テンプレート適用履歴（機能ID: 44, 62）
- **ページ:** `/architectures` → 「官公庁向け Web3層構成」クリック → `/architectures/tpl-1`
- **確認:** 「詳細」「適用履歴」の2タブ → 適用履歴タブで3件の履歴テーブル

## Task 8: デプロイキュー管理（機能ID: 58, 15）
- **ページ:** `/env/env-1/deploy-queue`
- **確認:** サイドバー「デプロイキュー」→ 3件のキュー（1件実行中、2件待機中）、「手動デプロイ」ボタン

## Task 9: Deployトリガー + Build設定（機能ID: 59, 60）
- **ページ:** `/env/env-1/deploy-trigger`
- **確認:** サイドバー「トリガー / ビルド」→ 2タブ切替
  - **トリガー設定タブ:** Webhookイベント3つのチェックボックス + ブランチ正規表現入力
  - **ビルド設定タブ:** 3サービスのビルドコマンド・出力先テーブル

## Task 10: 統合ログビューア（機能ID: 49, 50, 66, 67）
- **ページ:** `/env/env-1/logs`
- **確認:** サイドバー「ログ」→ 10件のログ一覧、右上ドロップダウンで「すべて/インフラ操作/デプロイ/システム」フィルタ切替

## 確認の流れ（推奨順序）

1. ログイン → `/workspace` → ワークスペース作成 (Task 1)
2. ヘッダーのベル → 通知一覧 (Task 2)
3. `/workspace/ws-1` → 環境カード「開発環境」クリック → `/env/env-1`
4. サイドバーで以下を巡回:
   - ドリフト検知 (Task 4)
   - リソース一覧 (Task 5)
   - Service / Infra (Task 6)
   - デプロイキュー (Task 8)
   - トリガー / ビルド (Task 9)
   - ログ (Task 10)
5. `/project/pj-1/settings` → GitHub連携 (Task 3)
6. `/architectures/tpl-1` → 適用履歴タブ (Task 7)
